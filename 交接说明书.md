# 奇瑞经销商网销点检项目 — 交接说明书

> 最后更新：2026-06-11
> 维护入口：本仓库 `xcbude/wxdj-2`，开发分支 `claude/kind-knuth-y3Dga`

---

## 0. 一句话说明

品牌方（奇瑞）定期核查各地经销商在**第三方汽车平台**上的网销价格执行情况：
批量抓取每家经销商页面的**店头名称、车型裸车价、最新软文日期、头图截图**，
再和官方下发的**报价标准**逐车型对比，价格不符的标红、标准里没有的车型标黄，
最终输出着色 Excel + 头图截图 ZIP，交给品牌方核查。

## 2. ⛔ 红线与注意事项（必读）

| 红线 | 说明 |
|------|------|
| **裸车价 0 容错** | 核价时裸车价必须**完全等于**标准值，差 1 元也标红。不许设容差 |
| **运行环境必须是中国大陆真实 IP** | 易车/汽车之家对云服务器 IP 做 WAF 封锁（`host_not_allowed`）。**必须在客户本地 Windows（家庭/企业宽带）跑**，云端服务器跑不通 |
| **单次任务只跑单个平台** | 一个任务只跑汽车之家**或**易车，不混跑。混跑任务过重、易中途失败白干 |
| **名单格式按表头自动识别** | 不许写死"第几行第几列"。客户可能插行/插列，必须用表头关键词定位 UID 列，否则一改表就拉取失败 |
| **不要尝试 MCP / GitHub 在客户侧连接** | 客户侧只交付可下载的文件（zip / xlsx / bat），不依赖 GitHub 连接 |
| **报价标准列含义固定** | 标准表 E列=市场价格、F列=对应媒体车型、H列=现金优惠（2026.6.2 那列）。裸车价=E−H |
| **截图/检查点不进 git** | `output*/` 下的截图和 checkpoint 已 gitignore |

## 4. 点检项目（采集字段）

两个平台采集项一致（易车输出项参考汽车之家）：

| 字段 | 说明 | 汽车之家来源 | 易车来源 |
|------|------|------------|---------|
| 省份 / 城市 | 经销商所在地 | 名单/简称推断 | 名单列 |
| 经销商 ID | 平台内部 UID | 名单 C 列 | 名单「易车UID」列 |
| 经销商名称 | 店头全称 | 页面 title 正则 | 页面 `<title>` 正则 |
| 车系 | 如「瑞虎8」 | 报价区段解析 | `cars.html` 解析 |
| 车型（车款） | 如「2026款 1.6T 双离合豪华型」 | 报价区段解析 | `cars.html` 解析 |
| 裸车价 | 本店实际报价 | 报价区段解析 | `cars.html` 本店报价 |
| 指导价 | 厂商指导价 | 报价区段解析 | `cars.html` 厂商指导价 |
| 最新软文日期 | 最近一篇促销新闻日期 | body 文本正则 | `news.html` 日期正则 |
| 头图截图 | 经销商主页 banner | Playwright 截图 | 下载 banner 图片 |

## 5. 不同平台的网页结构逻辑

### 5.1 汽车之家

```
经销商主页 URL：https://dealer.autohome.com.cn/{dealer_id}
报价页 URL：    主页 + 报价页后缀
```

- **店名**：页面 `<title>` 正则提取
- **报价**：Tailwind 动态类名，CSS 选择器不可靠 → `page.inner_text("body")` 取全文后**状态机解析**「车型报价」区段
- **展开全部车系**：JS 强制点击所有含「展开」文字的 button：
  ```js
  document.querySelectorAll('button').forEach(btn => {
      if (btn.textContent.includes('展开')) btn.click();
  });
  ```
- **反爬**：偶尔拦截（body < 200 字）→ 降并发到 1、加大 `REQUEST_DELAY`、断点重跑跳过已完成

### 5.2 易车（结构已摸清，纯静态 HTML）

```
经销商主页：  https://dealer.yiche.com/{uid}/
全部车型报价：https://dealer.yiche.com/{uid}/cars.html      ← 一次拿全部车系+全部车款
车系详情：    https://dealer.yiche.com/{uid}/cars_{车系ID}.html
车款详情：    https://dealer.yiche.com/{uid}/price_detail/{车款ID}.html
软文列表：    https://dealer.m.yiche.com/d{uid}/news.html
```

- **UID 格式**：9 位数字，`100XXXXXX`（字符串，非整数）
- **店名**：`<title>` → `【奇瑞广州品华体验中心4s店】...` 正则 `【(.+?)4[sS]店】`
- **`cars.html` 是关键页**：一个请求就含全部车系全部车款，格式（清理后文本）：
  ```
  瑞虎8 8.99-11.99 万 （厂商指导价：9.99-12.99 万）
    2026款 第五代 豹款 鲲鹏 1.6T 双离合豪华型  有现车 外观颜色  9.99万  ↓1.00万  8.99万  询价
    ↑车款名                                              ↑指导价   ↑优惠   ↑本店报价(裸车价)
  ```
  - **指导价 = 该行第一个 `X.XX万`**
  - **裸车价 = 该行最后一个 `X.XX万`（询价前）**
- **软文日期**：`news.html` 文本里 regex `20\d{2}-\d{2}-\d{2}`，取第一个（最新）
- **头图**：主页 banner `<img>`，下载为 PNG
- **反爬**：仅 IP 白名单（云服务器 403），本地宽带 IP 正常 200；TLSv1.3，requests 默认即可

## 6. 点检逻辑（核价匹配规则）

### 6.1 加载报价标准

读 `网销平台报价建议表` sheet（合并单元格需先解析填充）：
- E 列（col 5）= 市场价格（厂商指导价）
- F 列（col 6）= 对应媒体车型名
- H 列（col 8）= 现金优惠金额（2026.6.2 那列）
- **标准裸车价 = 市场价格 − 现金优惠**

### 6.2 匹配算法（按可靠性排序）

```
我方(车系, 车型, 指导价)
   ↓
① 车系映射 SERIES_MAP：我方车系名 → 标准车系候选列表
   ↓
② 在候选车系内，用「指导价 == 标准市场价格」唯一定位车型
   （指导价比车型名稳定，匹配率 94.5% vs 名称匹配 32%）
   ↓
③ 同一指导价命中多行 → 用车型名字符集 Jaccard 相似度消歧
```

### 6.3 着色规则

| 情况 | 着色 | 含义 |
|------|------|------|
| 裸车价 ≠ 标准裸车价（0 容错） | 🟥 裸车价单元格标红 | 价格执行不符 |
| 车系/车型无法在标准中匹配 | 🟨 车系单元格标黄 | 标准内无此车型，无法核对 |
| 裸车价 == 标准裸车价 | 不着色 | 合规 |

## 7. 处理规则补充

| 规则 | 说明 |
|------|------|
| **名单表头自动定位** | 扫描前 10 行找含「易车UID」的表头行；按关键词找列：`易车UID`→uid、`简称`→店名、`省`→省份、`城市/市`→城市。**插行插列不影响** |
| **UID 校验** | 易车 UID 必须匹配 `^1\d{8}$`（9 位）；汽车之家 UID 必须为纯数字。不符跳过 |
| **断点续跑** | 每完成一家追加一行 JSON 到 `checkpoint.jsonl`；重跑自动跳过已完成 |
| **价格单位** | 页面单位「万元」，核价时统一 ×10000 转元再比较 |
| **金额解析** | 正则 `([\d.]+)` 取数字，容错全角/空格 |
| **并发** | 汽车之家建议 1-3（浏览器重、易触发反爬）；易车可 3-5（纯 HTTP 轻量） |

## 8. 输出格式

| 文件 | 内容 |
|------|------|
| `output[_yiche]/result.xlsx` | 全量采集结果。每车款一行，经销商级字段（省/市/ID/名称/软文日期/截图）纵向合并，头图嵌入 |
| `output[_yiche]/result_checked.xlsx` | 上表 + 裸车价着色（红/黄）+「核对结果汇总」sheet（图例、统计、标红明细） |
| `output[_yiche]/头图截图.zip` | 各经销商头图 PNG，按经销商简称命名；缺失的列入 `【缺失说明】.txt` |
| `output[_yiche]/checkpoint.jsonl` | 断点文件（gitignore） |

**核对结果汇总 sheet 含**：图例、车型行总数、相符数、标红数、标黄数、全部标红明细行。

  yiche_probe.py / yiche_probe2.py / run_probe2.bat

## 10. 运行方式（客户侧，0 代码）

1. 解压工具包到一个文件夹
2. 把两个 xlsx（经销商名单、报价标准）复制进文件夹
3. **汽车之家**：双击 `启动点检任务.bat`
   **易车**：命令行 `python yiche_task.py`（或单测 `python yiche_test_one.py`）
4. 结果在 `output/`（汽车之家）或 `output_yiche/`（易车）

## 11. 已知坑与历史踩雷

| 坑 | 根因 | 解决 |
|----|------|------|
| `.bat` 乱码 | Linux 写的 UTF-8，Windows cmd 按 GBK 读 | bat 内容改纯 ASCII（文件名可中文） |
| `pip` 启动器报错 | 客户机 `pip.exe` 内置 python 路径失效 | 改 `python -m pip` |
| `pip install` 失败 | 连不上 PyPI | 改清华镜像源 + 官方回退 |
| Chromium 找不到 | 代码写死 Linux 路径 `/opt/pw-browsers/...` | `CHROMIUM_PATH=None`，回退系统 Edge/Chrome |
| 易车云端 403 | 平台 IP 白名单封锁云服务器 | 必须客户本地宽带 IP 运行 |
| task.py 改坏 | 客户手改误删字典键名 | 改为自动识别 xlsx，无需手改 |
| 名单插行错位 | 写死行列号 | 表头关键词自动定位 |
