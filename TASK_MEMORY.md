## 1. 任务目标

抓取 **汽车之家 (dealer.autohome.com.cn)** 网站内，**指定区域**的 **奇瑞 (Chery)** 经销商以下 4 个字段：

| 字段 | 含义 | 数据来源 | 采集方式 |
|------|------|---------|---------|
| 店头名称 | 经销商门店全称（如「风云东莞富瑞零售中心」） | 经销商首页 面包屑/标题 | 文本抓取 |
| 车型报价 | 该经销商在售车型 + 经销商报价（如 艾瑞泽8 8.99-12.89万） | 经销商首页「主推车型/车型列表」 | 文本抓取 |
| **头图（截图）** | 经销商首页顶部整块区域的**截图** | 经销商首页（见下「头图区域定义」） | **无头浏览器区域截图** |
| 最新软文发布日期 | 首页焦点图右侧资讯列表中**最新一条**的发布日期（如 2026-04-04） | 经销商首页 焦点图旁资讯列表 | 文本抓取取 max 日期 |

### 头图区域定义（截图范围）
经销商首页顶部一整块：从顶部导航栏（汽车之家 logo + 城市 + 搜索）→ 焦点图 banner → 导航 Tab（首页/车型列表/优惠促销…）→ 焦点大图 + 右侧软文列表 + 右侧门店信息卡。
即用户图二红框范围。需用无头浏览器渲染页面后，按元素/坐标裁剪截图。

### 软文定义（已确认）
= 经销商首页焦点图**右侧的资讯列表**（每条标题带日期，例：「风云T9L目前价格稳定 售价13.99万…」2026-04-04）。
「最新软文发布日期」= 该列表所有日期中的最大值。

## 2. 输入参数

- `region`：**广东 / 福建 / 海南 三省的所有市县**（全量，量大，预估数百家经销商）。
- `brand`：固定为「奇瑞」。
- MVP 阶段：先固定 **1 个城市 + 1 个经销商** 跑通，再逐城市扩展到三省全量。

## 3. 数据源线索（已知）

- 经销商首页 URL 形态：`https://dealer.autohome.com.cn/{dealerId}`（例 2207406）
- Tab 页：首页 / 车型列表 / 优惠促销 / 销售顾问 / 公司介绍 / 环境展示 / 智能展厅
- 店头名称：面包屑「当前位置：东莞 > 风云东莞富瑞零售中心 > 首页」
- 主营品牌：右下「主营品牌 - 奇瑞」（用于过滤奇瑞经销商）
- 待 Step0 确认：①按 省/市 + 奇瑞 brandId 查经销商列表的接口/页面；②软文列表是否在首页 HTML 内（看截图应是同页渲染）。

## 4. 输出要求（已确认）

- 格式：**CSV 或 .xlsx，且必须包含图片**。→ 选 **.xlsx**（CSV 无法内嵌图），用 `openpyxl` 把头图截图嵌入单元格。
- 每行一个经销商，列：`省, 市, dealerId, 店头名称, 车型报价(多车型合并), 最新软文日期, 头图截图(嵌入图片)`。
- 截图文件同时落盘到 `output/screenshots/{dealerId}.png` 作为备份。

## 5. 最小 MVP 拆解（自底向上跑通）

### Step 0 — 探源（一次性）
- 确认「省/市 + 奇瑞」查经销商列表的接口或页面 URL 规则 → `docs/endpoints.md`
- 确认经销商首页中：店头名称、车型报价、软文列表 的 DOM 选择器。
- 确认头图截图区域的元素选择器（红框对应的容器节点）。

### Step 1 — 经销商列表
- 输入 `省/市 + 奇瑞` → 输出 `[{省, 市, dealerId, 店头名称}]`。
- 验证：能打印某城市奇瑞经销商列表。

### Step 2 — 单店全字段（MVP 核心）
- 固定 1 个 dealerId，用无头浏览器打开首页：
  - 文本：店头名称、车型报价、最新软文日期
  - **截图**：裁剪红框区域 → png
- 验证：单店输出一条完整记录 + 一张截图。

### Step 3 — 批量（三省全量）
- 遍历三省所有市县 → 每市经销商列表 → 每店跑 Step2。
- 加：限速、失败重试、异常跳过、断点续跑（量大，必须可中断恢复）。

### Step 4 — 输出 xlsx
- 汇总数据 + 内嵌头图截图，导出 `.xlsx`。

## 6. 技术选型（因截图需求调整）

- 语言：Python 3
- **渲染+截图：Playwright（无头 Chromium）** —— 头图截图必须，软文/车型也顺带在渲染后的 DOM 取。
- 经销商列表：优先用 requests 走列表接口（若有）；否则 Playwright 翻列表页。
- 输出：`openpyxl`（.xlsx 内嵌图片）。
- 反爬最小应对：UA + 请求间隔 + 重试；量大时考虑代理（MVP 先不做）。
- 合规：仅在授权/合规范围采集，遵守站点条款与 robots。
